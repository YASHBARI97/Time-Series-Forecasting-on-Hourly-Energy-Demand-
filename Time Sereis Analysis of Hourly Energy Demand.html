from statsmodels.tsa.arima.model import ARIMA

# Ensure the index is unique before setting frequency
# This handles cases where original data might have duplicate timestamps
if not train_data.index.is_unique:
    print("Warning: Duplicate timestamps found in training data. Dropping duplicates.")
    train_data = train_data[~train_data.index.duplicated(keep='first')]

if not test_data.index.is_unique:
    print("Warning: Duplicate timestamps found in testing data. Dropping duplicates.")
    test_data = test_data[~test_data.index.duplicated(keep='first')]

# Set frequency for train_data and test_data explicitly to avoid warnings and ensure correct forecasting behavior
# Assuming hourly data, so freq='h'
train_data_sarima = train_data.asfreq('h')
test_data_sarima = test_data.asfreq('h')

# Define candidate SARIMA orders (p,d,q)(P,D,Q,s)
# Based on the best non-seasonal ARIMA(2,0,2) and considering daily seasonality (s=24)
candidates_sarima = [
    ((2,0,2), (1,0,0,24)),
    ((2,0,2), (0,0,1,24)),
    ((2,0,2), (1,0,1,24)),
    ((2,0,2), (0,1,0,24)),
    ((2,0,2), (0,1,1,24)),
    ((2,0,2), (1,1,0,24)),
    ((2,0,2), (1,1,1,24))
]

results_sarima = {}

print("Fitting SARIMA models...")
for order, seasonal_order in candidates_sarima:
    try:
        model_sarima = ARIMA(train_data_sarima, order=order, seasonal_order=seasonal_order)
        res_sarima = model_sarima.fit()
        results_sarima[(order, seasonal_order)] = res_sarima
        print(f"Fitted SARIMA{order}{seasonal_order} AIC={res_sarima.aic:.2f} BIC={res_sarima.bic:.2f}")
    except Exception as e:
        print(f"Order {order}{seasonal_order} failed: {e}")

# Pick best model by AIC
best_order_sarima = min(results_sarima, key=lambda o: results_sarima[o].aic)
best_res_sarima = results_sarima[best_order_sarima]
print("\nBest SARIMA order by AIC:", best_order_sarima)
print(best_res_sarima.summary())

The Ljung-Box test is implemented to check if the residuals of a time series model are random or if there is a significant autocorrelation remaining. This is crucial because a good model should explain all the predictable patterns in the data, leaving behind random noise (white noise). If the test reveals remaining autocorrelation, it indicates the model is not a good fit, and its predictions could be inaccurate.

Conclusion

Overall Project Goal: The objective was to perform a time series analysis of hourly energy demand to forecast future demand using ARIMA, SARIMA, and Prophet models.

Key Achievements:

Data Preparation: The hourly energy demand data was successfully loaded, meticulously preprocessed, and cleaned. This involved correctly combining date and time columns, setting a unique Datetime index, renaming the demand column to 'Demand', and robustly handling data quality issues such as invalid date entries (e.g., '29-Feb' in a non-leap year) by coercing and dropping problematic rows. The data was then split into training and testing sets (80/20). Stationarity Analysis: Initial checks using both Augmented Dickey-Fuller (ADF) and Kwiatkowski-Phillips-Schmidt-Shin (KPSS) tests confirmed that the 'Demand' time series is likely stationary, which is a prerequisite for many time series models. Autocorrelation (ACF) and Partial Autocorrelation (PACF) plots were used to understand the underlying patterns, indicating strong daily seasonality. SARIMA Model Implementation: A comprehensive SARIMA (Seasonal AutoRegressive Integrated Moving Average) modeling approach was successfully implemented. Several candidate SARIMA orders, considering a daily seasonality of 24 hours, were fitted to the training data. The model SARIMA((2, 0, 2), (1, 0, 0, 24)) was identified as the best-performing model based on the Akaike Information Criterion (AIC). Its performance was evaluated on the test set, yielding: Mean Squared Error (MSE): 1,756,645,962.548 Root Mean Squared Error (RMSE): 41,912.360 Mean Absolute Error (MAE): 34,437.247 A Ljung-Box test on the SARIMA model's residuals indicated that the residuals are largely random, suggesting the model has captured most of the underlying patterns, though some ConvergenceWarning and singular covariance matrix warnings were noted during fitting, suggesting potential model instability. ARIMA Model Implementation: Non-seasonal ARIMA models were also fitted and a best order of ARIMA(2,0,2) was identified based on BIC. While the models were fitted, a direct comparative plot of AIC/BIC for these models was not explicitly generated in the final sequence. Visualizations: Key visualizations were created, including the hourly demand over time, ACF/PACF plots for pattern identification, SARIMA model residuals, and a comparison of SARIMA forecasts against actuals, providing a clear visual assessment of the model's performance.
